ReasonBlocks is a Python SDK for production AI agents. It works with LangChain, the OpenAI Agents SDK, the Anthropic Messages API, and the Claude Agent SDK. Two capability areas are exposed across these frameworks:

- **Steering** — score each agent step, evaluate server-side trajectory monitors, inject E-trace guidance from a pattern store, and optionally route the model based on a difficulty FSM. The pipeline is shared via [`SteeringSession`](/api-reference/steering-session) and wired into each framework through a small adapter (LangChain middleware, Claude Messages loop helper, OpenAI Agents `Model` wrapper). The Claude Agent SDK path is telemetry-only because its agent loop runs inside the Claude Code CLI process.
- **Codebase knowledge** — `CodebaseMemory` for semantic per-repo findings storage and `ImportGraph` for Python blast-radius analysis. Both ship framework-specific tool factories (`make_langchain_tools`, `make_openai_tools`, `make_claude_tools`, `make_claude_agent_sdk_tools`) so the same memory layer plugs into whichever agent runtime you use.

## What works on which framework

| Capability | LangChain | OpenAI Agents | Claude Messages | Claude Agent SDK |
|---|---|---|---|---|
| Telemetry to the dashboard | yes | yes | yes | yes |
| FSM step scoring | yes | yes | yes | — |
| Server-side monitor steering | yes | yes | yes | — |
| E-trace injection (E1 / E2 / E3) | yes | yes | yes | — |
| Model routing | yes | yes (with factory) | yes | — |
| Token-saving compression + early exit | yes | — | — | — |
| `CodebaseMemory` tool factory | yes | yes | yes | yes |
| `ImportGraph` blast-radius queries | yes | yes | yes | yes |

The four entry points are:

- **`rb.middleware()`** — LangChain 1.0 `AgentMiddleware`. The reference implementation; full pipeline + token-saving + general-monitor middleware compose alongside it.
- **`rb.openai_model(default_model, ...)`** — wraps an `openai-agents` `Model` so `Agent(model=...)` runs the pipeline before each `get_response` call. Pair with an optional `model_factory` to enable model routing.
- **`rb.claude_messages_session()`** — builds a [`SteeringSession`](/api-reference/steering-session) you pass into [`run_messages_agent_loop(..., session=...)`](/api-reference/integrations/claude-tools#run_messages_agent_loop). The loop drives Anthropic's Messages API tool-use turns and runs steering at every turn.
- **`rb.claude_agent_telemetry()`** — telemetry-only adapter for the Claude Agent SDK `query()` stream. The agent loop is inside Claude Code, so steering can't apply, but per-tool `run_start` / `step` / `run_finish` events are emitted to the dashboard.

Token-saving compression + early exit are LangChain-only today (the implementation is itself an `AgentMiddleware`).

## What the steering pipeline does

On every step of a LangChain agent, the `ReasonBlocks` middleware:

1. Scores the agent's last reasoning step using a heuristic that combines hedging density, response length, error language, and entity density.
2. Advances a difficulty FSM (`INIT`, `FAST`, `NORMAL`, `SLOW`, `SKIP`) using the score plus recent history.
3. Posts the current trace to the ReasonBlocks API's `/monitors/evaluate` endpoint, which runs the monitor suite server-side and returns a steering intervention when the trajectory looks broken.
4. Retrieves up to three tiers of E-traces from the pattern store: E1 (instance-level, gated by monitor signal), E2 (failure-mode patterns), and E3 (universal rules, fired once on the first call).
5. Renders any pending injections into a `[REASONBLOCKS]` block appended to the system message, and overrides the model if `model_routing` maps the current FSM state to a different model.
6. Streams per-step telemetry to the ReasonBlocks dashboard.
